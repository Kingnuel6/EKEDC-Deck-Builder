# EKEDC Deck Builder — Claude Code Build Spec

## Project overview

Build a standalone web application called **EKEDC Deck Builder** for Eko Electricity Distribution PLC. The tool allows EKEDC managers to generate branded PowerPoint presentations (.pptx) in two ways: (1) filling a manual entry form, or (2) uploading a PDF or DOCX document where slide content has been pre-written and labelled.

The AI brain is powered by the OpenRouter API. The PPTX output is generated by PptxGenJS. There is no database and no authentication — this is a lightweight internal tool.

## Backend — server.js

Build an Express server with these routes:

### `GET /`
Serve `public/index.html`.

### `POST /api/generate/manual`
Accept JSON body:
```json
{
  "title": "string",
  "type": "string",
  "audience": "string",
  "presenter": "string",
  "keyMessage": "string",
  "slideCount": 8,
  "slideNotes": "string (optional, freeform slide-by-slide notes)",
  "style": "dark | light | accent"
}
```
Flow: call `ai.js` → get structured JSON → call `pptxBuilder.js` → return file download.

### `POST /api/generate/upload`
Accept `multipart/form-data` with:
- `file` — PDF or DOCX (via multer)
- `style` — string
- `presenter` — string

Flow: call `parser.js` to extract text → call `ai.js` → call `pptxBuilder.js` → return file download.

### `POST /api/parse-preview`
Same upload as above but only returns the detected slide structure JSON (no PPTX generated). Used by the frontend to show the "Detected slide structure" preview before the user confirms generation.

## File parser — parser.js

Export two async functions:

### `parsePDF(filePath)`
Use `pdf-parse` to extract full text from the PDF. Return the raw text string.

### `parseDOCX(filePath)`
Use `mammoth` to extract raw text from the DOCX. Return the raw text string.

Export a router function `extractText(filePath, mimetype)` that calls the right parser based on mimetype.

## AI module — ai.js

### `generateSlides(input)`

`input` is either:
- `{ mode: 'manual', data: { title, type, audience, presenter, keyMessage, slideCount, slideNotes } }`
- `{ mode: 'upload', text: '<extracted document text>', style, presenter }`

Make a POST request to:
```
https://openrouter.ai/api/v1/chat/completions
```

Headers:
```
Authorization: Bearer ${OPENROUTER_API_KEY}
Content-Type: application/json
HTTP-Referer: http://localhost:3000
X-Title: EKEDC Deck Builder
```

Model: `anthropic/claude-3.5-sonnet`

**System prompt (use exactly this):**

```
You are a presentation architect for Eko Electricity Distribution PLC (EKEDC), a Nigerian electricity distribution company serving Lagos and Ogun States.

Your job is to return a valid JSON array of slide objects. No markdown, no explanation, no code fences — ONLY the JSON array.

Each slide object must follow this exact schema:
{
  "slideNumber": 1,
  "layout": "title" | "content" | "two-column" | "stat-callout" | "section-divider" | "closing",
  "title": "string",
  "subtitle": "string (optional, for title/section-divider slides)",
  "bullets": ["string", "string"] (optional, for content/two-column slides),
  "leftColumn": { "heading": "string", "bullets": ["string"] } (optional, for two-column),
  "rightColumn": { "heading": "string", "bullets": ["string"] } (optional, for two-column),
  "stats": [{ "value": "string", "label": "string" }] (optional, for stat-callout — max 3 stats),
  "speakerNotes": "string (2-3 sentences the presenter can say for this slide)"
}

Rules:
- Always start with a layout: "title" slide
- Always end with a layout: "closing" slide
- Use "stat-callout" for any slide with key numbers or KPIs
- Use "section-divider" to break up long decks into chapters
- Keep bullet text under 12 words per bullet
- Speaker notes should be natural, professional Nigerian business English
- If slide content is sparse or missing, infer sensible EKEDC-appropriate content from context
- Return ONLY the JSON array. Nothing else.
```

**User message for manual mode:**
```
Create a ${slideCount}-slide presentation for EKEDC with the following details:

Title: ${title}
Type: ${type}
Audience: ${audience}
Presenter: ${presenter}
Key message: ${keyMessage}
Slide notes: ${slideNotes || 'None provided — use your best judgment.'}

Return the JSON array of slides.
```

**User message for upload mode:**
```
The following is the text content extracted from a document uploaded by an EKEDC manager. 
They have outlined their slide content. Parse it into a structured slide array.
If slides are labelled ("Slide 1:", "Slide 2:", etc.), respect that structure exactly.
If content is sparse for any slide, expand it intelligently using context from the rest of the document.

DOCUMENT TEXT:
${text}

Presenter: ${presenter || 'EKEDC Management'}

Return the JSON array of slides.
```

Parse the response: strip any accidental markdown fences, then `JSON.parse()` the result. If parsing fails, retry once with a stricter prompt. If it fails again, return a 500 error with message "AI failed to generate valid slide structure."

## PPTX builder — pptxBuilder.js

Use PptxGenJS to build the presentation. Accept the slides JSON array and a `style` parameter (`dark`, `light`, or `accent`).

### Brand tokens

```js
const BRAND = {
  navy:   '#1A1A8C',
  yellow: '#FFD700',
  red:    '#CC0000',
  white:  '#FFFFFF',
  lightBg:'#F4F6FB',
  darkText:'#1A1A8C',
  bodyText:'#333333',
}
```

### Slide dimensions
Use widescreen (13.33 x 7.5 inches). Set in PptxGenJS as `pptx.defineLayout({ name: 'WIDE', width: 13.33, height: 7.5 })`.

### Style themes

**dark** (navy background, white text, yellow accents):
- Background: `#1A1A8C`
- Title text: `#FFFFFF`
- Body text: `#FFFFFF`
- Accent color: `#FFD700`
- Use for: title slide, section-dividers, closing slide

**light** (white background, navy text, yellow accents):
- Background: `#FFFFFF`
- Title text: `#1A1A8C`
- Body text: `#333333`
- Accent color: `#FFD700`
- Use for: content slides, two-column slides

**accent** (yellow highlight bar on navy):
- For stat-callout slides: white background, stat values in `#FFD700` on `#1A1A8C` badges

### Slide layouts to implement

#### `title`
- Full navy background
- EKEDC bolt emoji replaced by a yellow rectangle shape (20pt wide, 4pt tall) as accent bar above title
- Title text: white, 40pt, bold, centered
- Subtitle text: white, 18pt, normal, centered
- Presenter name bottom-left in white 12pt
- Date bottom-right in white 12pt (use `new Date().toLocaleDateString('en-NG', { month: 'long', year: 'numeric' })`)
- "Eko Electricity Distribution PLC" footer centered at bottom in yellow 10pt

#### `content`
- White background
- Navy left accent bar (0.1" wide, full height) on left edge
- Title: navy, 28pt, bold, top
- Bullets: dark text, 14pt, with yellow circle bullet markers
- Footer on every slide: "Eko Electricity Distribution PLC | Confidential" in navy 9pt, bottom center
- Slide number bottom-right in navy 9pt

#### `two-column`
- White background
- Title: navy, 24pt, bold, top
- Left column with heading (navy, 14pt bold) + bullets
- Right column with heading (navy, 14pt bold) + bullets
- Thin vertical divider in yellow between columns
- Standard footer

#### `stat-callout`
- White background
- Title: navy, 24pt, bold
- Up to 3 stat boxes side by side, each: navy background, stat value in yellow 36pt bold, label in white 12pt below
- Standard footer

#### `section-divider`
- Full navy background
- Large title text centered, white, 36pt bold
- Thin yellow horizontal line above title (decorative shape, 6" wide, 0.05" tall)
- No footer needed

#### `closing`
- Full navy background
- "Thank You" or custom title in white 40pt bold centered
- Subtitle in white 18pt centered
- Yellow accent bar shape
- "Eko Electricity Distribution PLC" in yellow 14pt centered below
- Website: `www.ekedp.com` in white 11pt

### Output
Save the file as `output/EKEDC_Presentation_${Date.now()}.pptx` and return the file path. The Express route will then `res.download()` the file and delete it after 60 seconds using `setTimeout(() => fs.unlinkSync(filePath), 60000)`.

## Frontend — public/index.html

### Layout
- Fixed top navbar (56px height) in `#1A1A8C`
  - Left: yellow lightning bolt icon + "EKEDC Deck Builder" title + "Eko Electricity Distribution PLC" subtitle
  - Right: "OpenRouter connected" green status badge + Settings button (non-functional for now)
- Two-column layout below navbar:
  - Left sidebar (280px wide, white bg, border-right): navigation between Manual Entry and Upload Document modes + Recent Decks list (hardcoded placeholder items) + Brand palette swatches
  - Main content area: the active form panel

### Mode toggle
Clicking "Manual entry" or "Upload document" in the sidebar switches the main content area. Also implement as tab buttons at the top of the content area. Use vanilla JS to show/hide the panels — no page reload.

### Manual entry panel
Form fields (all use Tailwind classes for styling):
1. Presentation title (text input, required)
2. Presentation type (select dropdown): Board/Executive briefing, KPI/Performance report, Project update, Staff briefing, Outage/Incident report, Customer comms
3. Target audience (text input)
4. Presenter name (text input)
5. Key message / objective (textarea, required)
6. Slide count selector: pill buttons for 5, 8, 10, 12, Custom. Custom reveals a number input.
7. Slide-by-slide content (textarea, optional): placeholder text shows the "Slide 1: ..., Slide 2: ..." format example
8. Slide style selector: three visual cards — "Executive dark", "Clean light", "Bold accent"

Buttons at bottom:
- "Generate presentation" (primary, navy bg) — submits the form
- "Preview outline first" (secondary) — calls `/api/parse-preview` equivalent for manual mode, shows a modal with the AI-structured outline before generating PPTX

### Upload document panel
1. Drag-and-drop zone with dashed border. Accepts .pdf and .docx only. On file drop/select, show the file name + size.
2. After file is selected, auto-call `POST /api/parse-preview` to detect slide structure and display the "Detected slide structure" preview list below the drop zone.
3. Each detected slide shows: slide number badge (navy), slide title, bullet preview, and a status dot (green = content rich, yellow = sparse).
4. Style selector (dropdown) and Presenter name (text input) below the preview.
5. Info strip: "Slides with sparse content (yellow dot) will be intelligently expanded by AI..."
6. "Generate from document" button (primary) + "Edit slide structure" button (ghost)

### Loading state
When generation is in progress:
- Disable the generate button
- Replace button text with a spinner + "Generating your deck..."
- Show a progress bar animation below the button (fake progress — animate from 0% to 85% over 30 seconds, then jump to 100% when response comes back)
- Show estimated time "~25–40 seconds" next to the button

### Success state
When the file is ready:
- Show a success banner: green checkmark + "Your presentation is ready!" + "Download .pptx" button
- The download button should trigger the file download via `window.location.href = '/download/filename.pptx'` or use the blob URL from the response
- Show a "Generate another" button to reset the form

### Error state
If the API returns an error:
- Show a red error banner with the error message
- Keep the form filled so the user doesn't lose their work

## API endpoints summary

| Method | Path | Purpose |
|---|---|---|
| GET | `/` | Serve frontend |
| POST | `/api/generate/manual` | Generate PPTX from form data |
| POST | `/api/generate/upload` | Generate PPTX from uploaded file |
| POST | `/api/parse-preview` | Extract + preview slide structure only |
| GET | `/download/:filename` | Serve generated PPTX file |

## Error handling

- If OpenRouter API key is missing: return 500 with "OpenRouter API key not configured"
- If uploaded file is not PDF or DOCX: return 400 with "Only .pdf and .docx files are supported"
- If file is larger than 10MB: return 400 with "File too large. Maximum size is 10MB"
- If AI returns invalid JSON after 2 retries: return 500 with descriptive error
- All errors must be shown in the UI as a dismissable red banner — never a raw browser alert

## Implementation notes for Claude Code

- Build all files in the order: `package.json` → `src/server.js` → `src/parser.js` → `src/ai.js` → `src/pptxBuilder.js` → `public/index.html` → `public/main.js` → `README.md`
- After writing each file, run `node -e "require('./src/[filename]')"` to check for syntax errors before moving on
- After all files are written, run `npm install` and then `node src/server.js` to verify the server starts without errors
- Do not use TypeScript — plain JavaScript only
- Do not use any frontend framework — vanilla HTML/CSS/JS only, Tailwind via CDN
- The Tailwind CDN script tag: `<script src="https://cdn.tailwindcss.com"></script>`
- multer storage config: use `diskStorage` with destination `./uploads/` and preserve original filename with a timestamp prefix
- Clean up uploaded files after processing: `fs.unlinkSync(filePath)` in a finally block
- PptxGenJS must be imported as: `const PptxGenJS = require('pptxgenjs')`
- Test the full flow manually before declaring complete: upload a sample PDF, verify slide detection, verify PPTX downloads and opens in PowerPoint/LibreOffice
